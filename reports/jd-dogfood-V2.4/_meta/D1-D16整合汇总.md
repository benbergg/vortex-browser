## V2.4 真实瓶颈 (重新识别)

按"前置自检回路" — V2.4 修复后**未达成** spec 目标 30s, **真因**:

1. **通信固定开销 ~23s (主要)**: MCP client ↔ vortex-server 通信, 不受 auto-wait/maxElements/fill 影响
2. **page-side scan 3-30ms**: 几乎不耗时
3. **京东 React Fiber 持续重渲染**: ref hash 每次 observe 都变, 浪费 observe 时间
4. **京东 SPA root delegation 拦截**: 跳转未解决

## V2.4 进一步优化方向 (P1+)

按"前置自检回路" + 评测数据 — **下一步优化** 需针对**通信开销**:

### P1.1 通信 batch
- vortex_observe + vortex_fill + vortex_act 合并为 1 次 RPC
- 减少 MCP ↔ vortex-server round-trip
- 预计: 总流程 94s → 50s

### P1.2 减少 bbox 序列化
- observe 返回时, bbox 默认 omit (只发 ref + name)
- 用户需 bbox 时再 includeBoxes=true
- 预计: 通信数据量 -80% (100 KB → 20 KB)

### P1.3 京东 SPA root delegation 适配
- vortex_act click 增加 "preHover" 模式
- hover 200ms → mousedown → mouseup → click 序列
- 预计: 京东跳转成功率 0% → 80%

### P2.1 maxElements 默认进一步减少
- 默认 80 → 50 (顶部 50 元素, 跨平台)
- 预计: 通信数据量 -40%
