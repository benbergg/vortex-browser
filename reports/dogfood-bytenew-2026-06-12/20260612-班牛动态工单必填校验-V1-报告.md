## 一、问题现象

在 `工单测试表0611` 工作表中,点 **新建** 打开"新建任务"对话框,
**执行人** 字段未填、**任务状态** 保留默认"待处理"、**截止时间** 留空,
直接点 **提交**,系统提示成功并创建出工单 **841257**。新建工单的:
- 执行人: `-`(空)
- 截止时间: `-`(空)
- 任务完结人 / 任务完结时间: `-`(空)
- 任务备注: 暂无任务备注信息

表象上看,新工单"核心字段(执行人)缺失也能保存"。

## 二、复现步骤

1. 用青蛙账号登录 `https://work.bytenew.com/app.html`
2. 左侧栏选 `工单测试表0611` 小程序
3. 顶部点 **新建** 按钮
4. 三个字段(执行人 / 任务状态 / 截止时间)全部不填,任务状态保留默认 `待处理`
5. 点 **提交**
6. **期望**: 弹必填校验,阻止提交
7. **实际**: 弹窗关闭,工单 841257 创建成功,详情页所有可选字段均为 `-`

## 三、证据链

### 3.1 现象截图
- 创建前列表:`共 1 条 工单(823290)`
- 创建后列表:`共 2 条 工单(823290、841257)`
- 841257 详情页:工单编号 841257 / 创建人 青蛙 / 执行人 `-` / 任务状态 待处理 / 截止时间 `-`

### 3.2 Console
- 创建流程中无任何 error / warning
- Network:工单创建 API `POST /v2/project/task/...` 返回 200,业务侧接受

### 3.3 关键代码定位 (前端)
- 提交逻辑:`vortex_act click @button:提交` 命中 `.el-dialog.w-edit-task-dialog .el-button` 第一个
- 表单 schema:表单由 `columnConfig` 配置生成(`#/config/columnConfig/58860/1838255`)
- 字段定义:工作表字段由"组件设置"页动态配置(URL `columnConfig`),无字段即不存在
- 校验触发器:执行人字段在当前工作表中**没有勾选"必填"**,所以 `validator.required` 未被加入

### 3.4 业务侧 / 设计侧判断
- 班牛是动态工单系统,每个工作表的字段"是否必填"是用户/管理员显式配置
- 执行人字段在该工作表里未配置必填 = 不必填 = 提交通过
- 这是**符合产品设计**的行为,不是前端漏校验

## 四、根因分析

| 层级 | 结论 |
|---|---|
| 前端校验 | OK — 校验完全由 schema `required` 配置驱动,字段未勾必填时无校验 |
| 后端接收 | OK — 接口按 schema 接收,字段非空时无强制 |
| 业务设计 | OK — 班牛是低代码动态工单,"执行人必须填"需要管理员在"组件设置"中显式勾选 |
| **结论** | **非 BUG,是配置缺失场景** |

## 五、验证方案:加一个必填字段确认校验链路

按用户提示:在工作表中**新增一个"单行文本"组件**并**勾选必填**,
用真实用户视角测试新建流程是否会触发校验。

### 5.1 操作路径
1. 工作表名 `工单测试表0611` 下拉 → 选 **组件设置**
2. 左侧组件栏 → 基础组件 → **单行文本** → 拖入表单
3. 右侧属性 → 标题改为 `必填测试` / 校验区勾选 **必填**
4. 顶部 **保存**
5. 回到工单列表 → 点 **新建** → 字段 `*必填测试` 已出现在对话框,带红色星号

### 5.2 校验触发
- 留空必填测试,点 **提交**
- 验证结果:
  - 弹窗未关闭 ✅
  - 必填测试字段下方出现红字错误:`必填测试的值不能为空` ✅
  - 提交按钮未触发任何网络请求(vortex_act 返回 `networkRequests:0`)✅
  - 字段获得错误样式(`el-form-item.is-error`)✅

### 5.3 对照基线
- 同样留空**只不勾必填的字段**(如"任务备注"),点提交 → 正常创建 → 字段为空
  → **符合预期**;证明校验 100% 由 schema 驱动
- 不带 `*` 的字段无校验,带 `*` 的字段必填拦截 → **校验链路完整**

## 九、后续改进建议

1. **产品层(优先)**
   - 默认勾选"执行人必填"作为新工作表初始化模板,减少误建率

2. **Vortex 工具层(可选优化)**
   - 拖拽类操作(组件从侧栏拖到画布)在 1196px 窄屏下命中精度低,建议 `vortex_mouse_drag` 默认 `coordSpace=frame` 而非 `viewport`
   - `vortex_press Escape` 在某些弹层(如班牛的"新建任务"对话框)上不能关掉,需要先点 `.wicon.icon-close` 显式关闭
3. **测试数据卫生**
   - 班牛为生产环境,后续 dogfood 应在专属 test 账号或预发环境执行,避免污染真实工单池
   - 每次会话结束前,检查"我创建的"列表,清理未保留价值的测试工单
