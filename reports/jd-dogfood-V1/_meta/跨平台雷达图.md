## 3. 关键洞察

### 3.1 京东 vs 淘宝空名率都 ~30% 但根因完全不同

| 平台 | 空名 `<a>` 实体 | textContent 状态 | d4b7330 适用性 |
|------|----------------|-------------------|----------------|
| 淘宝 | 整张商品卡 `<a class="doubleCardWrapperAdapt">`(235×374) | 含完整商品信息(¥/标题/销量/回头客) | 修法方向错(看 directTextNodes 而非 textContent) |
| 京东 | 客服图标 `_customer_service_icon_zclqt_60`(16×16) + 京东 logo(190×58) | 空字符串(本来就无信息) | 与京东**无关**(京东商品卡不是 `<a>`,无 textContent 信息) |

**结论:** 淘宝需要"重新提取"(改 textContent 判定),京东需要"合成兜底名"或"识别后过滤"。两者修复路径完全不同,d4b7330 这一 commit 对京东根本不适用,需独立 patch(参考 `_meta/P1-1京东根因诊断.md` §5.4 方案 A+B 组合)。

### 3.2 P1-2 修复未在京东泛化

3 品类(3C / 家电 / 服饰)与淘宝 V4 逐字一致:NOT_STABLE 实际经 `auto-wait.ts:89` 抛出时用 `TIMEOUT` 错误码,而 518d500 修复的是 `errors.hints.ts` 的 NOT_STABLE hint —— 永远不触发。修复路径仍需 N0059-V5 重做(`auto-wait.ts:89` 改用 NOT_STABLE 码当 `lastReason === NOT_STABLE`)。

### 3.3 P2 修复泛化生效

京东 description IIFE 提示与淘宝同 PASS:`"MAIN world. async=fn body, IIFE. No cross-origin iframe."` 在两平台 3 品类均生效,且 vortex_evaluate 在京东 3 品类详情页函数表达式包裹后均能正常求值。**这是本次 V1 验证唯一全 PASS 的修复点。**

### 3.4 京东独有 BUG 密度高于淘宝

| 平台 | 评测周期 | 新立项 BUG/REQ | 关键根因 |
|------|----------|----------------|----------|
| 淘宝 N0059-V3 | V3 | 5 个独立缺陷 | 通用 observe / actionability 缺陷 |
| 淘宝 N0059-V4 | V4 | BUG-008 + REQ-009(2 个) | sticky bar CTA + IIFE 描述 |
| 京东本次 V1 | V1 | BUG-009/010/011/012/013/014 + REQ-NNN(7 个) | 京东 SPA 化 + React 重写 + 虚拟列表 + 客服独立站 |

**京东 BUG 密度 (7) > 淘宝 V4 (2)** 主要源于京东深度 React + SPA 化,vortex 现有 observe / actionability 策略对淘宝传统 SSR 友好但在京东命中率下降。

### 3.5 京东 React 重写 + react-virtuoso 虚拟列表是京东独有技术挑战

- 京东 React 重写后,大量商品卡使用 `<div onClick>` + `cursor:pointer`,observe 漏抓(BUG-010)
- 京东商品列表使用 react-virtuoso 虚拟滚动,popper 瞬态覆盖触发 OCCLUDED(BUG-012)
- 淘宝目前仍以传统 SSR 为主,这两个挑战不存在或较弱

### 3.6 跨 SKU = 整页 SPA 跳转是京东 SPA 性能模式

服饰品类切颜色/尺码时,京东会触发整页 SPA 重新挂载,vortex ref 跨 SKU 失效是**设计选择**(非 BUG)。LLM 需在跨 SKU 后重新 observe 获取新 ref,这是预期行为。

## 5. 平台能力差异总结

| 维度 | 淘宝特点 | 京东特点 |
|------|----------|----------|
| UI 模式 | 双列商品卡 + sticky bar | 多列商品卡 + 价格区 CSS transition |
| 渲染 | SSR 立即渲染 | SPA 懒加载(空调规格表需滚动触发) |
| 客服 | 智能助手浮窗(可能含 iframe) | 独立 jdcs.jd.com 新 tab(0 iframe) |
| 配送 | 地址选择 + 时段 grid | 静态计算文字(无 grid, **BUG-014 业务侧已评估**: 不补, 8 维度 D3 改 N/A, 详见 _meta/BUG-014 §7) |
| SKU 切换 | 单 SKU 切换 | 颜色×尺码二维库存矩阵 |
| 跨 SKU | 整页跳转 | 整页跳转(同) |
| 价保 | (淘宝基本无该独立模块) | 静态文字 + help.jd.com 跳转 |
| 物流服务 | (淘宝多样,但分散) | 9 种物流方式 + 6 种服务标签(全部 help.jd.com) |
| CTA 元素 | `<div class="btnItem">` 不是 `<button>`(BUG-008) | React `<div onClick>` + cursor:pointer(BUG-010) |
| 表单字段 | 标准 `<input>` 为主 | 标准 `<input>` 为主,但 vortex_fill 缺 force(BUG-011 共性) |
| 虚拟列表 | (淘宝较少) | react-virtuoso(BUG-012 独有) |
