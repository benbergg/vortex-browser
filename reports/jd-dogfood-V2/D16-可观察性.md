## 5. 真 gap #1 (ROI 最高): vortex_debug_read.filter 子字段未文档化

### 5.1 现象 (V2 重测仍复现)

**实测** (淘宝家电详情, V2 沿用):
```bash
# 测试 1: filter.urlPattern 字段名错误
vortex_debug_read source=network filter='{urlPattern: "item.taobao.com", statusMin: 200, statusMax: 299}'
→ Error: vortex_debug_read source=network: pattern is required (pass top-level 'pattern' or 'filter.pattern')

# 测试 2: filter.pattern 字段名正确 (按错误提示)
vortex_debug_read source=network filter='{pattern: "taobao.com"}'
→ ✅ 生效
```

**正确用法** (实测):
```bash
# 顶层 pattern 字段 (必填, handler 要求)
vortex_debug_read source=network pattern='s.taobao.com' statusMin=200 statusMax=299
→ 多个商品 API 200 OK ✅
```

**console 错误实测** (V2 沿用):
```bash
vortex_debug_read source=console filter='{level: "error"}' tail=5
→ 0-1 个错误 (filter.level 生效) ✅
```

### 5.2 根因 (源码 + schema 双重失配)

- **`schemas-public.ts:201` filter 字段无 description**:
  ```ts
  filter: { type: "object" },  // ❌ 无 description
  ```
- **`console.ts:160-163` handler 已实现 `args.level` 过滤**:
  ```ts
  const level = args.level as string | undefined;
  logs = logs.filter((l) => l.level === level);
  ```
- **`network.ts:305-321` handler 已实现复合过滤** (字段名不统一):
  ```ts
  const urlPattern = args.url as string | undefined;       // line 305 用 args.url
  const statusMin = args.statusMin as number | undefined;   // ✅ 一致
  const statusMax = args.statusMax as number | undefined;   // ✅ 一致
  ```
  ```ts
  const urlPattern = args.urlPattern as string | undefined;  // line 253 用 args.urlPattern
  ```
  **line 253 vs line 305 字段名不统一** (`url` vs `urlPattern`), 但实际 handler 错误信息说要 `pattern`:
  ```
  "pattern is required (pass top-level 'pattern' or 'filter.pattern')"
  ```
  → 实际**第三个**字段名 `pattern` 才被接受。源码混乱, schema 又未声明, LLM 完全无法靠文档查。

### 5.3 修复建议 (一行 description 修复, ROI 最高)

**P0 候选**: 修改 `packages/mcp/src/tools/schemas-public.ts:201`:
```ts
filter: {
  type: "object",
  description: "console: {level:'error'|'warn'|'all'}; network: {pattern:'<substr>', statusMin:200, statusMax:299}",
  // 注: 实际 handler 字段名 'pattern' (不是 'urlPattern'), 需源码先统一
}
```

**同时修 handler 字段名** (handler 三处 `urlPattern`/`url`/`pattern` 不统一, 应统一为 `pattern`)

**工作量估**: 0.05d (一行 description) + 0.1d (handler 字段名统一)
