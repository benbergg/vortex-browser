## 1. 维度概述

## 2. 评测方法

- **主路径** (Opus 订正): `vortex_evaluate` 读 `aria-*` / `role` / `document.activeElement` (拆 `ariaLabelCoverage` vs `accessibleNameCoverage` 两指标, 不被文本兜底掩盖)
- **兜底**: `vortex_evaluate` 替代 `vortex_observe` 抓 a11y props (因 observe 不可用, 降级)
- **pass 判据**:
  - ✅ PASS: `ariaLabelCoverage` ≥ 90%
  - ⚠️ PARTIAL: 50-90%
  - ❌ FAIL: <50%
  - `accessibleNameCoverage` 作旁证记录
- **人化路径** (按 `ecom-anti-detection` skill):
  - 京东首页 (`www.jd.com`) → 真实搜索 (input value setter + Enter) → 真实点击商品卡 (`vortex_mouse_drag` 真实 mouse event, 触发 SPA 跳转)
  - 跨品类 navigate 搜索页 (类目切换合法) + 真实点击
  - 评价/加购页: 详情页内**真实点击**评价 tab → 测 aria → 滚到加购按钮 → 测 aria
  - **评价区用 `._closeIcon_1ygkr_39` 关闭按钮** 显式关闭 (避免状态卫生问题)
  - 跨域间隔 8-10s, 跨品类 10-12s, 状态稳定等待 3-5s

## 4. 工具能力 gap 诊断

### 4.1 真 gap #1: vortex_observe 在 main frame 0 全部扫空 (原语层能力缺失, ROI 最高)

**现象**:
```
vortex_observe scope=viewport/full filter=interactive/all
  → "# frame 0 not scanned (url=...)"
```

**根因** (debug_read console error 实证, V2 重测仍复现):
```
ReferenceError: applyReactClickableMarker is not defined
    at <anonymous>:1:14814
    at <anonymous>:1:15441
```

**源码分析**:
- `packages/extension/src/handlers/observe.ts:228` `applyReactClickableMarker` 是 handler 顶层 export 函数
- 在 `scanOneFrame` (observe.ts:254) 用 `chrome.scripting.executeScript` 注入 page-side MAIN world 时, **`func` 参数被序列化为字符串** (background scope 中定义的 `applyReactClickableMarker` / `REACT_CLICKABLE_HINT` 不进字符串)
- dist build (`background.ts--x5mOpJ-.js`) 验证: 函数定义在 background scope, 引用在 inject func body 内, **序列化时引用对象未带过去**
- 结果: page-side MAIN world 抛 ReferenceError → `scanOneFrame` 返 null → "frame 0 not scanned"

**实测覆盖** (V2 重测确认, 跨双平台):
- search.jd.com ❌
- item.jd.com ❌
- s.taobao.com ❌ (淘宝同症状)
- example.com ❌ (验证非京东反爬 / X-Frame-Options 问题)
- 双平台一致 → **不是** 平台问题, **是** vortex 自身 bug

**V2 重测新增影响**:
- D9 用 evaluate 替代 observe 抓 a11y, 工作量略增但数据真实
- D11 BUG-012 修复验证**仍**用 fixture 回归锁, 沿用 V1 (`93f60fb`)
- D3 主路径交互评测**受 observe 不可用影响最大**, 但 V2 不重测 D3 (V1 已闭环)

**修复建议** (V2.1 候选, 真原语层, 详见 `工具能力gap汇总.md`):
- 方案 A: 在 page-side inject func 内**内联** `applyReactClickableMarker` 逻辑 (0.3d, 最简)
- 方案 B: 移到 page-side 模块通过 `loadPageSideModule` 注入
- 方案 C: 改 dist build 让 `applyReactClickableMarker` 真 inline

### 4.2 真 gap #2: 缺 vortex_a11y_audit 工具 (审计层空白, 定位待定)

**现象**:
- 京东**搜索/列表/详情/评价/加购** 全 5 阶段 aria 4.76-15.15% — **京东 a11y 系统性差**
- 缺一个工具能**扫整页 a11y 缺失** (类似 axe-core), 当前需手动 `vortex_evaluate` 逐个读
- 建议: 新增 `vortex_a11y_audit` 工具, 扫所有 `[role]` / `[aria-*]` 缺失, 输出 axe-style JSON 报告
- 注: 审计套件层, **非原语层**, 是否做取决于 vortex 定位

## 6. 风险点 + 真站 vs fixture 边界

- **真站 vs fixture**: 京东真站 a11y 表现差, 不需 fixture 复现; 但 V2.1 修 observe 需用 fixture 验证
- **跨平台对比**: 京东详情 aria (~14%) > 淘宝详情 (~1%); 淘宝搜索 (~10%) > 京东搜索 (~6%) — 双平台各有所长, **都差**
- **诚实标注**: D9 15 格子全 FAIL 是真结论 (京东 aria < 16%), **不是** 评测方法问题
- **路径选择原因** (按 `ecom-anti-detection` skill):
  - **绝对不**直接 navigate `item.jd.com/<id>.html` (冷跳 = 风控元凶)
  - 跨品类: 搜索页 navigate (类目切换合法, 京东不重定向)

  - 商品卡真实 mouse_drag 触发 SPA 跳转 (anti-detection L1, isTrusted=true)
  - 评价区用 `._closeIcon_1ygkr_39` 显式关闭 (anti-detection L1 状态卫生)
