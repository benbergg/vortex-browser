## 二、问题 1 — 列统计缺少空值/无数字段保护

### 2.1 现象

在没有任何数值/金额字段的工作表上点 **列统计** → 弹出层只有 **求和 / 求平均** 两个选项,无提示文字告知"当前工作表无数值/金额字段"。

### 2.2 复现步骤

1. 进入 `工单测试表0611` (字段:执行人/任务状态/截止时间/任务完结人/任务完结时间 + 新加的 `必填测试` 文本字段,**全为非数值字段**)
2. 点列上的 **列统计** 按钮
3. 弹层出现,显示 `统计数值、金额组件的值` 提示 + 求和/求平均

### 2.3 证据

- 弹层文字:`统计数值、金额组件的值` 仅一行
- 没有任何 `disabled` 的状态显示给求和/求平均
- 点击求和/求平均:实际行为未知(未测试,因为会产生无效结果)

### 2.4 根因分析

| 层级 | 结论 |
|---|---|
| 前端 | 弹层只判断"是否有统计",未根据实际字段类型 disable 选项 |
| 后端 | 接受统计请求并返回 0 或 null |
| 设计 | **不友好**:用户在 0 数值字段工作表上点了列统计,应明确告知"无可统计字段" |
| **结论** | **轻微 UX 问题,非阻塞 BUG** |

### 2.5 建议

- 弹层加空态:`当前工作表无数值/金额字段,请添加后重试`
- 求和/求平均按钮 `disabled` 当无可统计字段
- 数据导出列名提示可加 `建议:导出时增加统计列(数量/合计)`

### 2.6 状态

- 截图:已记录(本轮)
- 网络:无异常请求
- Console:无 error/warn
- 修复:无需代码修复(产品/UX 改进)

## 三、问题 2 — 流程编辑器节点连线交互异常(潜在 BUG)

### 3.1 现象

在流程编辑器中,选中节点后能看到 4 个蓝色连接点(上/下/左/右),但从一点拖到另一点,**没有连线被创建**。LogicFlow (`.lf-graph`) 是底层库。

### 3.2 复现步骤

1. `工单测试表0611` → 流程 → **流程设置**
2. 拖入 `审批` 节点,再撤销
3. 拖入 `判断` 节点,撤销
4. 选中 **流程发起** 节点 → 4 个蓝点出现
5. 从 流程发起 的 **底部蓝点** 拖到 **流程结束** 的 **顶部**
6. **期望**:画出带箭头的连线
7. **实际**:无连线,节点仍独立

### 3.3 证据(代码定位)

- 画布实现:`<div class="lf-graph">` 内嵌 LogicFlow(锚点类 `lf-anchor`、边类 `lf-edge`)
- 节点结构:`<foreignObject><div class="wf-custom-node2 wf-start-node2">`
- 选中节点后,DOM 增加 4 个锚点 `circle/rect`(实测未找到,可能由 SVG `<g>` 内部渲染)
- `vortex_mouse_drag` 多次尝试(包括 25 步慢速、垂直 200px 距离、25px 步进),均未生成 `.lf-edge` 元素

### 3.4 根因分析

| 可能 | 结论 |
|---|---|
| LogicFlow 不支持 mousedown→mousemove→mouseup 创建边 | ✗ 它支持(标准库) |
| Vortex `mouse_drag` 用 CDP `Input.dispatchMouseEvent`,跟 LogicFlow 监听的 pointer/pointerup 事件不匹配 | **高概率** |
| 流程编辑器的"边"创建有特定手势(例如先点 dot 再画 Bezier) | 可能在某些版本中 |
| 班牛二次封装覆盖了默认拖拽逻辑 | 中等概率 |
| **结论** | **可能是 Vortex 工具的拖拽实现与 LogicFlow 期望的事件序列不匹配** |

### 3.5 证据细节

- `vortex_mouse_drag fromX=1020,fromY=480,toX=1020,toY=678 steps=15` 后:
  - `networkRequests: 0` (无网络)
  - `domMutations: 0` (无 DOM 变更)
  - 节点仍独立,无 `.lf-edge` 元素生成
- LogicFlow 的 `addEdge({sourceNodeId, targetNodeId})` API 应该是无 drag 也能用的(但需 Vue 组件暴露)
- Canvas 画布 transform:`matrix(1,0,0,1,-722,361)`,节点在 graph (180, 76) 处

### 3.6 修复方案

**Vortex 端(推荐先做)**:
- 给 `vortex_mouse_drag` 增加 `force: true` 选项,允许自定义事件序列
- 新增 `vortex_workflow_connect` 工具,专门处理 LogicFlow 这类流程编辑器的边创建
- 文档化:哪些场景下 `mouse_drag` 会失败,提示用户

**班牛端(待产品评估)**:
- 流程编辑器的边创建交互是否对所有鼠标设备一致?Safari/touch 是否兼容?
- 节点连接是否有键盘快捷键(例如选中节点按 L 弹连线菜单)?

### 3.7 状态

- 截图:已记录(选中和拖拽前后对照)
- 网络:无
- Console:无 error,只有 `getPullConfigurationRules` 等业务日志
- 修复:**建议在 Vortex 端改进**(`mouse_drag` 增强或专用工具);不需要改 班牛

## 四、问题 3 — 流程编辑器节点无属性面板

### 4.1 现象

在流程编辑器中,点击/双击/右键 节点,均**不打开任何属性配置面板**。这与主流流程编辑器(Vue Flow、react-flow、LogicFlow demo)的设计不符。

### 4.2 复现步骤

1. 拖入 `审批` 节点到画布
2. 单击/双击/右键 节点(在节点容器或图标上)
3. **期望**:右侧或弹层打开节点属性(审批人、审批方式、抄送人等)
4. **实际**:节点高亮(选中状态)但无属性面板

### 4.3 证据

- DOM 扫描:无 `.el-drawer`、无 `.node-config-panel`、无 `aside` 出现
- CSS 扫描:无 `[class*=properties]`、`[class*=config-panel]`、`[class*=node-detail]` 显示
- 顶部三个 tab 是页面级(组件设置/流程设置/工单功能),非节点级

### 4.4 根因分析

| 层级 | 结论 |
|---|---|
| LogicFlow 原生 | 提供 `node.db.click` / `node.db.dblclick` 回调,需业务方注册 |
| 班牛封装 | 看起来**只用了单击选中**,未实现属性编辑 UI |
| **结论** | **设计缺失,不是 BUG,是功能未完成** |

### 4.5 修复方案

**Vortex 端**:
- 文档化:流程编辑器节点属性配置需用其他方式
- 不需要 Vortex 修复

**班牛端(产品/前端)**:
- 增加节点属性配置 UI
- 在 `组件设置`/`流程设置` tab 之外,或用右侧 drawer 模式
- 至少右键节点出菜单:删除/复制/属性

### 4.6 状态

- 截图:已记录
- 修复:**Vortex 无需改**;**班牛功能补全**,但优先级低(可能是设计意图,通过 `更多设置` 单独配置)

4. **期望**:toast 提示"发布成功"或"请先连接节点"等
5. **实际**:页面无变化,无反馈

### 5.3 证据

| 项 | 值 |
|---|---|

| 网络请求 URL 匹配 | 无明显 `publishWorkflow`/`deployWorkflow` 端点(可能用通用 `/workflowConfig/save` 或类似) |
| 节点状态变化 | 无 |
| DOM 变化 | 0 |
| 弹窗/toast | 0 |
| Console | 无 publish 相关日志,只有 `getPullConfigurationRules` 业务调用 |

### 5.4 根因分析

| 层级 | 结论 |
|---|---|
| 前端 | publish 按钮的 click handler 缺少 loading/success/error 反馈 |
| 后端 | 可能返回 200 但未检查业务码 |
| 设计 | 流程状态变更**不反馈**给用户,违反基本 UX 原则 |
| **结论** | **明显 BUG,应优先修复** |

### 5.5 修复方案

**Vortex 端**:
- 不需要修复(Vortex 操作端已经看到点击生效)

**班牛端(高优先级)**:

- 后端返回后弹 toast:
  - 成功:`发布成功!当前生效版本:v1.x (YYYY-MM-DD HH:mm)`
  - 失败:`发布失败:XXX 原因`
- 错误码对应不同错误原因
- 未连接的节点:发布时高亮 + 提示"存在 X 个未连接节点"

## 六、问题 5 — 节点删除需通过 Undo,Vortex 键盘 Delete 失效

### 6.1 现象

选中节点后按 **Delete** 或 **Backspace** 键,**不删除节点**。但 顶部 Undo 按钮(撤销)能删除。

### 6.2 复现步骤

1. 拖入 `审批` 节点
2. 单击选中(虚线边框)
3. 按 **Delete** 键
4. **期望**:节点消失
5. **实际**:节点仍在

### 6.3 证据

- `vortex_press key=Delete` → `success: true, focusedElement: div`
- `vortex_press key=Backspace` → 同上
- DOM 检查:节点 DOM 仍在
- 只有 `wf-undo-btn` 撤销按钮有效

### 6.4 根因分析

| 层级 | 结论 |
|---|---|
| LogicFlow | 支持 `delete` / `backspace` 键删除选中节点(默认快捷键) |
| 班牛封装 | 可能未在画布根元素上挂 keydown 监听,或被外层 wrapper 截获 |
| 焦点 | 选中节点后焦点在 `div` 上,而非 canvas |
| **结论** | **UX 缺陷,用户难以理解为何删除键无效** |

### 6.5 修复方案

**Vortex 端**:
- 文档化:流程编辑器节点需用 Undo 删除

**班牛端(中优先级)**:
- 在 canvas 容器上挂 `keydown`,识别 `Delete`/`Backspace`/`Backspace` 触发节点删除
- 至少显示工具提示告知用户删除方式

### 6.6 状态

- 截图:已记录
- 影响:中,用户需要靠 撤销 按钮来回退
- 修复:**班牛端 P2 级**

## 七、问题 6 — LogicFlow 画布的 viewport 平移/缩放状态机不稳

### 7.1 现象

当画布发生意外平移(我测试时 mouse_drag 误触了画布),节点的 transform matrix 变成 `matrix(1,0,0,1,-722,361)`,节点飞到屏幕外,且 LogicFlow 自动恢复 transform 后,`适应屏幕` 按钮(适应屏幕)有时不能把节点拉回视野。

### 7.2 复现步骤

1. 进入流程编辑器
2. 在画布空白区域按住拖动(误触或主动)
3. 节点被平移出可见区域
4. 点 `适应屏幕` 工具栏按钮
5. **期望**:画布自动 fit 全部节点
6. **实际**:节点仍不可见,需要手动再用 wheel 缩放

### 7.3 证据

- transform: `matrix(1,0,0,1,-722,361)` (节点应在屏幕 y=76+361=437,但 x=180+(-722)=-542 在屏外)
- 调用 `适应屏幕` 工具栏按钮:节点仍在屏外
- 节点当前 visible boundingRect: x=-542, y=487 (屏外)

### 7.4 根因分析

| 层级 | 结论 |
|---|---|
| LogicFlow | `fitView` API 存在,但似乎没在 适应屏幕 按钮上正确调用 |
| 班牛封装 | 适应屏幕 按钮可能调用了错误的方法或参数 |
| 用户操作 | 用户用 `vortex_mouse_drag` 误触了画布,canvas 接受为 panning 事件 |
| **结论** | **班牛工具栏的"适应屏幕"功能不工作** |

### 7.5 修复方案

**Vortex 端**:
- 文档化:流程编辑器画布的 pan 操作可能破坏视图
- `vortex_mouse_drag` 工具增加 `skipPannableSelectors`,自动跳过 LogicFlow 画布

**班牛端(低优先级)**:
- 验证 `适应屏幕` 按钮是否真正调用 `lf.fitView()`
- 修复 LogicFlow 版本兼容性

### 7.6 状态

- 截图:已记录
- 影响:中,可通过刷新页面恢复
- 修复:**班牛端 P3 级**

## 八、问题 7(非 BUG,观察)— 流程编辑器的"开始"和"结束"节点内置隐式连接?

### 8.1 观察

在工作流设置页面(workflowMixSetting),`流程发起` 和 `流程结束` 节点默认存在。它们之间没有显示连接线,但根据 LogicFlow 默认行为,工作流的有效性可能要求:
- 流程发起 → 至少一个操作节点 → 流程结束
- 即:`流程发起.out` → `someNode.in` → `someNode.out` → `流程结束.in`

### 8.2 推论

如果用户从未画线,理论上流程可能不是有效工作流。`流程发布` 也许会因此静默失败但无错误提示(见问题 5)。

### 8.3 建议

- 班牛应在画布渲染时绘制一条半透明虚线提示"建议连接 流程发起 到 流程结束"
- 发布时若检测到未连接,显示"存在 X 个未连接节点,仍要发布吗?"确认弹窗

## 十四、关键发现(补充,2026-06-12 21:42)

### 14.1 Vue 班牛 + LogicFlow 架构剖析

通过 `vortex_evaluate` 注入的 BFS 探针发现:

```
workflow-config (Vue 2 root, __vue__)
└── addWorkflow
    ├── BnWorkFlow
    │   └── lf: LogicFlow 实例 (graphModel, viewMap, history)
    └── ...其他子组件
```

关键字段:
- `BnWorkFlow.lf` 直接暴露 LogicFlow 完整实例
- `addWorkflow.taskFlowData`: Vue 班牛持有的"权威"图数据(发往后端)
- `addWorkflow.isClickSave` / `loadingSave` / `showPublishBtn` / `showSaveBtn`: 按钮状态机
- `addWorkflow.saveProcess` / `releaseWorkFlow` / `validateWorkflowProcess`: 业务方法(都是 native code 编译后)

### 14.2 LogicFlow API 100% 可用,UI 渲染完美

```js
const lf = bnInst.lf;  // LogicFlow 实例
lf.addEdge({
  sourceNodeId: '2ndvhqlu0ys000',  // 流程发起
  targetNodeId: '50b55596-81c9-44c3-b3c0-e74252b799c5',  // 判断
  type: 'polyline'
});
lf.addEdge({
  sourceNodeId: '50b55596-81c9-44c3-b3c0-e74252b799c5',
  targetNodeId: '4ggm4vff556000',  // 流程结束
  type: 'polyline'
});
// → 立即在画布看到两条带箭头的折线,自检按钮从 disabled 变 enabled
```

**这彻底推翻了"LogicFlow 拖拽不工作 = 班牛 功能缺陷"的假设**:
- 班牛后端 / 数据层完全 OK
- LogicFlow 库完全 OK
- **Vortex 的 mouse_drag 与 LogicFlow 期望的 PointerEvent 序列不兼容** —— 这是 Vortex 工具层问题

### 14.3 🚨 严重架构隐患:Vue 班牛 与 LogicFlow 双 state 失同步

调用 `lf.addEdge` 后立即查 `addWInst.taskFlowData.edges`:

```json
"edges": []
```

**LogicFlow 内部 graph 已有 2 条 edge,Vue 班牛 taskFlowData 还是空数组!**

后果链:
1. 班牛`保存`按钮: `showSaveBtn: false` → DOM 显示 disabled,无法点击保存
2. 班牛`流程发布`按钮: 显示 enabled,但发布的是 `taskFlowData`(空 edges)
3. 即使用户手动点击 发布,后端会收到"无连线的图" —— 等同于发布了一个空流程
4. 班牛的 `lf.on('edge:add', ...)` 事件监听器本应同步两边 state,**这是必经路径**

**这意味着**:
- 班牛 流程编辑器 **不允许通过任何 API 路径(脚本、键盘)添加节点/边**,只允许拖拽
- 班牛 团队可能在 `lf.on('element:add', syncTaskFlowData)` 监听器里有 bug,或者从未考虑过非交互修改
- **AI Agent、自动化测试、无障碍读屏、键盘导航** 等场景全部失效

### 14.4 流程发布按钮仍无视觉反馈(补充验证)

手动同步 taskFlowData 后调用 `addWInst.saveProcess()`:
- 返回 `undefined`
- `isClickSave` 仍为 `false`
- `loadingSave` 仍为 `false`
- 无任何网络请求
- 无 toast / dialog

再次点击 `流程发布` 按钮(已经手动同步过的 graph):
- 0 network requests
- 66 DOM mutations(基本是 Vue render)
- 无任何视觉反馈

**结论:流程发布按钮的 click handler 缺少 loading/success/error 任何反馈,严重 UX 缺陷** —— 这是产品 BUG,优先级 P0。

### 14.5 改进建议升级

| P | 项 | 描述 | 类别 |
|---|---|---|---|
| **P0** | 流程发布反馈 | 缺 loading/success/error toast,用户无法确认结果 | 班牛 BUG |
| **P0** | Vue+LogicFlow state 同步 | 班牛禁止任何非交互修改,辅助技术/自动化失效 | 班牛 架构 |
| **P1** | 节点属性配置 UI | 缺属性面板 | 班牛 功能缺失 |
| **P1** | 节点删除键盘 | Delete/Backspace 无效 | 班牛 UX |
| **P1** | Vortex `mouse_drag` 适配 LogicFlow | 增加 `force: true` 或 pointer 事件序列参数 | Vortex 工具 |
| **P2** | Vortex 增加 `vortex_workflow_connect` 工具 | 抽象 LogicFlow 边创建,统一 API | Vortex 工具 |
| **P2** | 适应屏幕 按钮 | 不工作 | 班牛 BUG |
| **P2** | 未连接节点提示 | 发布时检查 | 班牛 UX |
| **P3** | 列统计空态 | 弹层加空态 | 班牛 UX |
| **P3** | 节点右键菜单 | 删除/复制/属性 | 班牛 UX |

```js
const root = document.querySelector('.workflow-config').__vue__;
// BFS find BnWorkFlow
const q = [root]; const seen = new Set();
let bn; while (q.length) {

const lf = bn.lf;

const start = data.nodes.find(n => n.type === 'StartNode2');
const end = data.nodes.find(n => n.type === 'EndNode2');

lf.addEdge({ sourceNodeId: start.id, targetNodeId: middle.id, type: 'polyline' });
lf.addEdge({ sourceNodeId: middle.id, targetNodeId: end.id, type: 'polyline' });
// 验证 LogicFlow 内部 graph
console.log(lf.getGraphData().edges);  // 2 条
// 验证 Vue 班牛 data 不同步
const addW = bn.$parent; // 找 addWorkflow
console.log(addW.taskFlowData.edges);  // [] ← 关键!
```
