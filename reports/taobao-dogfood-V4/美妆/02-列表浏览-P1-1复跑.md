## 3. 代码定位

**d4b7330 P1-1 修复**（需读 patch 确认精确行号）：
```ts
// d4b7330 在 isContainer 判定前加:
const hasDirectText = Array.from(el.childNodes).some(
  n => n.nodeType === Node.TEXT_NODE && n.textContent.trim().length > 0
);
if (hasDirectText) {
  return normName(el.textContent);
}
// 退化: 无直属文本再走 isContainer
```

**问题**:
1. 淘宝商品卡 `directTextNodes === []` → `hasDirectText === false` → 走 isContainer → 仍判空名
2. textContent 包含完整商品信息（标题/价格/销量），但修复不看 textContent
3. **修复方向错** — 应该"判 textContent 是否含商品特征"（如 ¥、人付款、回头客），而不是"判直属文本节点"

## 4. 优化技术方案

### 4.1 推荐方案 A — 判 textContent 含商品特征

```ts
// observe.ts 替代 d4b7330 的 hasDirectText 判定
const PRODUCT_HINTS = /[¥￥]\d|人付款|回头客|已售|月销|\d+\.\d+元/;
const text = normName(el.textContent);
if (text && PRODUCT_HINTS.test(text)) {
  return text;  // 含商品特征, 用自身文本
}
if (text && !isContainer) return text;
// 退化
```

**优势**:
- 直接命中 textContent 信息（标题/价格/销量）
- 不依赖 DOM 结构（图片主导卡片、纯文本卡片都覆盖）
- 不需要合成兜底名（信息最丰富）

### 4.2 备选方案 B — 仿 V3 §4 方案 2 合成 `product-card @x,y`

```ts
if (isContainer && /Card|Adapt/.test(el.className)) {
  return `product-card @x=${bbox.x},y=${bbox.y}`;
}
```

**优势**: 简单可靠
**劣势**: 信息密度低，LLM 仍要 click 进入详情才能知道是什么商品

### 4.3 推荐合并方案 A+B

方案 A 命中商品卡带价格的情况，方案 B 兜底不带的（如部分促销卡片）。
