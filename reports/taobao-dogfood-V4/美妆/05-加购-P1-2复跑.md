**响应**:

**结论**: 修复后 hint 不含 `force=true` 关键词。**518d500 修复对实际 NOT_STABLE 抛出路径未生效**。

### 1.3 根因诊断

**518d500 修的代码**（`packages/shared/src/errors.hints.ts:152-158`）:
```ts
NOT_STABLE: {
  hint: "Element position is unstable. If it sits inside a sticky/fixed ancestor with a CSS transition (e.g. `transition: bottom 0.15s`), retry vortex_act with options: { force: true } to bypass the stability check (CDP realMouse). Otherwise call vortex_wait_for mode='idle' to let the animation settle.",
  recoverable: true,
},
```

**518d500 加的测试**（`packages/shared/tests/errors.test.ts:124-145`）:
```ts
it("NOT_STABLE hint 提及 sticky/fixed/transition 容器场景", () => {
  const hint = DEFAULT_ERROR_META.NOT_STABLE.hint;
  const coversContainer = /sticky|fixed|position:\s*(sticky|fixed)|ancestor/i.test(hint);
  const coversTransition = /transition|animation|CSS/i.test(hint);
  expect(coversContainer).toBe(true);
  expect(coversTransition).toBe(true);
});

it("NOT_STABLE hint 显式建议 force=true 兜底", () => {
  const hint = DEFAULT_ERROR_META.NOT_STABLE.hint;
  expect(hint).toMatch(/force\s*[:=]\s*true|force=true/);
});
```

**但实测** `auto-wait.ts:89-96` 抛出的错误是:
```ts
throw vtxError(
  VtxErrorCode.TIMEOUT,        // ← 用了 TIMEOUT 错误码,不是 NOT_STABLE!
  `Actionability timeout after ${timeout}ms; last reason: ${lastReason ?? "unknown"}`,
  { selector, extras: { lastReason, ...(lastExtras ?? {}) } }
);
```

**关键诊断**:
- `auto-wait.ts` 抛 `VtxErrorCode.TIMEOUT` + 自定义 message
- 错误码是 `TIMEOUT` 不是 `NOT_STABLE`
- vortex MCP 端用 `TIMEOUT` 错误码查 `errors.hints.ts` 拿 hint（**不是** `NOT_STABLE` 的 hint）
- 所以 `errors.hints.ts` 的 `NOT_STABLE` hint 永远不被触发
- **518d500 修复路径选错**：应该改 `auto-wait.ts` 用 `VtxErrorCode.NOT_STABLE` 抛错，或把 TIMEOUT 的 hint 加上 lastReason=NOT_STABLE 时的 force=true 建议

### 1.4 fix 路径（推荐）

**方案 A（首选）**: `auto-wait.ts` 改用 NOT_STABLE 错误码

```ts
// auto-wait.ts:88-96
const lastReasonIsStability = lastReason === "NOT_STABLE";
throw vtxError(
  lastReasonIsStability ? VtxErrorCode.NOT_STABLE : VtxErrorCode.TIMEOUT,
  lastReasonIsStability
    ? `Element not stable after ${timeout}ms (last reason: NOT_STABLE)`
    : `Actionability timeout after ${timeout}ms; last reason: ${lastReason ?? "unknown"}`,
  { selector, extras: { lastReason, ...(lastExtras ?? {}) } }
);
```

**方案 B（备选）**: 在 TIMEOUT hint 拼接 lastReason 上下文

```ts
// errors.hints.ts TIMEOUT hint 扩展:
TIMEOUT: {
  hint: "Action timed out. Increase the timeout argument, or call vortex_wait_for with mode='idle' to let the page settle. If the timeout was caused by NOT_STABLE in a sticky/fixed ancestor with CSS transition, retry with options: { force: true } to bypass stability check.",
  recoverable: true,
}
```

**方案 A 优势**: 与 V3 §4 P1-2 残留方案精确对应，518d500 已加的 2 个测试可直接复用
**方案 B 优势**: 不动 auto-wait.ts 行为，单独扩 TIMEOUT hint

|------|------|------|-------|

### 1.6 兜底验证

**force=true 兜底**:
```

```

**响应**: `{"success": true, "element": {"tag": "span", "text": "领券购买"}, "x": 1297.1875, "y": 1218, "mode": "realMouse"}` ✅

**关键观察**:
- 兜底**成功** — `force=true` 确实能让 vortex_act 跳过 stability check 走 CDP realMouse

- 这是 V4 P1-2 修复**必须改进**的核心：让 LLM 一次看明白

## 2. vortex_observe 漏 sticky bar — 独立 BUG 发现

**问题**: 淘宝详情页 sticky bottom action bar (`Actions--aqEomNIE`) 内的"领券购买"/"加入购物车"/"立即购买"按钮：
- 是 `<div>` 容器，**不是** `<button>` 或 `<a>`
- 含 `<i>` icon + `<span>` 文本
- 内部有 8 个可点击子元素（被 .tb-toolkit-new 容器包含）

**observe 行为**:
- `scope=viewport, filter=interactive`: 完全没出现 sticky bar
- `scope=full, filter=interactive`: 同样没出现
- `scope=full, filter=all`: e74 "领券购买" span 出现，**但无 bbox**，无法 click

**根本原因**（推测）:
- filter=interactive 默认只看 `<a>/<button>/<input>/<select>/<textarea>/[role=button]>` 等标准控件
- `<div>` 即使是淘宝的核心 CTA（带 `onclick` / `style="background:..."`），也被过滤
- vortex 自身没有淘宝特定启发式规则

**V4 发现的新需求**:
- observe 应识别"高对比度背景 + click handler + 居中文本"作为可交互候选
- 或支持用户自定义 heuristic filter

**严重度**: 🟠 P1 — 评测主路径核心 CTA 找不到，破坏 V4 评测价值

**记录**: 见 §5 行动项 BUG-008
