vortex_debug_read(network) roundtrip

Clicking Fire requests fires two fetch() calls to unique-named relative URLs. Both 404 on the playground server — that's intentional: the extension's Network domain logs requests by URL + method regardless of status, and 404s are the cheapest stable way to produce reproducible entries without depending on a real API server. After both promises resolve, the result region updates so the bench case knows the fetches actually completed.
