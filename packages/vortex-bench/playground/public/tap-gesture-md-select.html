Tap-gesture target (BUG 8 regression fixture)

What this fixture exercises: the button below only responds to a real tap gesture, which is mousedown + mouseup within a short window. A bare click event (such as what HTMLElement.click() dispatches) is intentionally swallowed via event.stopPropagation() + event.preventDefault() — this mirrors the gesture recognisers used by AngularJS Material $mdGesture, Hammer.js, and pre-v3 Element / Ant Select widgets, all of which block lone clicks to avoid duplicate triggers on touch + mouse devices.

Before v0.8.2, vortex_act action="click" dispatched only el.click(), so this fixture would never report value=tapped. The fix dispatches the full pointerdown → mousedown → pointerup → mouseup → click sequence.
